# AI 辅助对话 Log（整理版）

> 说明：本 log 为课程提交用整理版，保留关键决策和代码生成过程，不逐字还原所有对话。

## 1. 需求确认

用户：我要做一个 Python 大作业，主题是手写数字识别。要求有 UI、注册登录、历史记录、关于页面、环境配置和说明文档。

AI：建议使用 Web UI + Flask + PyTorch CNN。Web 前端更适合实现手写画布、实时概率条形图、登录注册和历史页面。

## 2. UI 方案

用户：左侧需要可以鼠标手写，右侧实时显示 0-9 的概率分布，效果类似截图。

AI：设计为实验台界面。左侧使用 HTML Canvas 绘制白色笔迹，右侧用原生 JavaScript 绘制概率柱形图。用户每次移动鼠标后，把画布缩放为 28x28 灰度数组发送到 `/api/predict`。

## 3. 模型方案

用户：希望有训练数据集、训练器、原始模型和可直接使用的 demo。

AI：建议训练脚本与 demo 分离。`train.py` 负责下载 MNIST 并训练 CNN；`app.py` 默认加载已训练权重，保证用户可以直接使用。如果权重不存在，则提示先运行训练脚本。

## 4. 代码生成

AI：生成 Flask 路由、SQLite 用户表和历史表、密码哈希逻辑、PyTorch CNN 模型、训练循环、推理预处理、Canvas 前端交互、页面模板和 CSS。

用户：确认按方案实现。

AI：实现项目文件，并补充 README、AI 使用说明和课程报告草稿。

## 5. 测试建议

AI：建议测试注册登录、画布识别、历史记录隔离、关于页面、训练脚本和 README 中的启动命令。
